## 프로그램 설치과정 ##
1. 우선 https://github.com/ 에 가입한 다음, 카톡방에 자기 id를 올린다.
2. apmsetup을 설치한다. 설치는 http://moonwal.kr/318를 참고한다. 최종버전은 APMSETUP 7 201001030 임.
3. msysgit을 설치한다. 설치는 http://git-scm.com/에서. 최신 안정화 버전은 2.1.3버전.
	-> 꼭 이거 말고도 GitHub for Windows 등 예쁜 프로그램 많은데, 아마 이런거 쓰면 암걸릴테니 그냥 이거 깔고 콘솔창에서 작업하는걸 강력하게 추천함. 정말로.
4. 메모장이나 서브라임 텍스트 등 자기가 원하는 문서 편집 프로그램을 준비한다.


## 세팅 ##
5. apmsetup 트레이 아이콘을 우클릭 -> 서버 환경 설정을 누른다
6. MYSQL 설정 탭에서 신규 계정 생성을 누름
7. 위에서부터 차례대로 아래처럼 씀. 현재 root 계정 비빌번호는 기본값은 apmsetup임

	현재 root 계정 패스워드: apmsetup
	MySQL 계정아이디: dbproj
	MySQL 패스워드: dbproj
	패스워드 재입력: dbproj
	생성할 DB이름: dbproj

	나머지는 아무것도 안 건들고 신규 계정 생성 누름
8. Apache 설정 탭에서 DocumentRoot 폴더 위치를 확인.


## git 세팅 ##
	!! 여기부터는 대연이한테 권한 설정을 받은 다음에 진행할 것 !!
	(위의 3에서 말한 콘솔용 git을 기준으로 설명함. 다른 프로그램도 다 같은 기능이 있으니깐 재주껏 똑같이 하면됨)

9. 위에 8번에서 찾은 DocumentRoot에 써있던 폴더를 탐색기에서 연다.
10. 폴더에서 마우스 우클릭 한다음 git bash를 눌러 콘솔창을 연다.
11. git init 이라고 입력하고 엔터. 이 폴더에서 앞으로 git을 사용할 것이라는 뜻임.
12. 다음과 같이 입력해서 서버를 등록함.
	git remote add origin https://github.com/ikasty/DBTeamProj.git
	이렇게 하면 origin 이라는 이름으로 위 서버가 내 컴퓨터에 등록됨.
13. 서버의 데이터를 다음과 같이 입력해서 받아옴.
	git pull origin master
	origin 서버에서 master 브랜치(메인 브랜치)를 가져오라는 뜻.
	아마 처음 하면 github 아이디와 비밀번호를 입력하라는 메시지가 뜰 것임. 오류 나면 두 번 정도 더 해볼 것.
14. 사용자 설정을 함
	git config --global user.name 홍길동
	git config --global user.email sample@sample.com
	git config --global push.default simple
	처음 두 개는 이름과 이메일 주소를 설정하는 것. 이름 대신 아이디를 사용해도 됨.
	마지막 거는 서버에 push할 때(업로드할 때) 오류를 방지하기 위함임.


######### 실제 작업 과정 #########
예를 들어, 로그인 기능을 만든다고 하자.

1. 다음과 같이 입력해서 브랜치를 새로 만듬
	git checkout -b 브랜치이름
-b는 새로 만들라는 뜻임. 여기서는 로그인을 만드는 거니깐, 이름을 이런 식으로 만든다.
	git checkout -b dev/login
dev/login이 이름임. 폴더처럼 / 로 구분도 할 수 있음.

2. 다음과 같이 입력해서 현재 상황을 볼 수 있음. 수시로 입력해보기 바람.
	git status

3. 필요한 기능을 구현함. 자기 컴퓨터에 APMSETUP을 깔았으니까, 구현이 제대로 됬는지는 http://localhost 에 접속해서 보면 됨.

4. 구현하는 중간 중간마다 다음과 같이 쳐서 기록함
	git add .
	git commit -m "로그인 창을 만듬"
git add . 는 수정된 모든 파일을 저장할 것이라고 선언하는 것. 마침표(.) 대신 파일 이름을 써서 각각 선택할 수도 있음.
선택된 파일 현황은 git status로 확인 가능함.
그리고 마지막 줄로 로그인 창을 만들었다는 메시지와 함께 현재 하던 내용을 저장하는거임.

5. 구현이 완성되었다면, 메인 시스템에 반영해야 한다. 메인 시스템의 브랜치는 master임.
	git checkout master
	git pull origin master
	git merge dev/login
브랜치를 master로 이동한 다음, 데이터를 불러와서, dev/login을 메인 시스템(master)에 합친다는 뜻.

6. 이제 서버에 변경한 내용을 업로드한다.
	git push origin master
origin이라는 서버(위 git 세팅의 12번 참조)에 master 브랜치를 업로드한다는 뜻임.
!!! 만약 업로드가 안된다면, 내가 작업하는 사이에 누가 먼저 올려서 충돌이 났다는 뜻임.
당황하지 말고 아래처럼 작업하자.
	git fetch origin master
	git merge origin/master
	git push origin master
http://git-scm.com/book/ko/v1/분산 환경에서의 Git 프로젝트에 기여하기의 '비공개 소규모 팀'을 참조한다.

7. 만약 다른 사람들도 같이 봐야 할 브랜치라면, 그 브랜치를 업로드할 수도 있음.
	git push origin dev/login
dev/login이 중요하다고 판단해, 서버에 올려서 다른 사람들과 공유하게 됨.

8. 만약 더 이상 dev/login이 필요 없을거 같으면 이 브랜치를 지움(변경 기록 등은 그대로 다 남아있음)
	git branch -d dev/login
